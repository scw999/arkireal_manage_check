S-1

권위 기반 인바운드 퍼널

대표님은 이미 상담 없이 신뢰를 만드는 자산(저서·유튜브·방송·칼럼)을 갖고 있습니다. 이걸 퍼널로 연결하면, 진성 고객이 이미 알고·신뢰하고·공부하고·래포를 쌓은 상태로 도착합니다. 목표는 상담 건수를 늘리는 게 아니라, 데워진 고객만 상담에 도달시켜 대표님의 시간 투입을 최소화하는 것입니다.

STAGE 1

인지

  • 저서 「집짓기 진실」
  • 유튜브 밈건축가
  • 방송 tvN·KBS2
  • 칼럼·강연
STAGE 2

교육·자기선별

  • 문제해결 콘텐츠
  • 무료 진단 상품
  • "내 문제가 뭔지 알겠다"
STAGE 3

래포

  • 진단 결과 리포트
  • 후속 메시지
  • "내 케이스를 이해한다"
STAGE 4

상담

  • 데워진 고객만 도착
  • 대표 시간 최소
  • 설득 → 확정
STAGE 5

계약

  • 제안서 발송
  • 설계·시공 진행
핵심 전환 원리: 각 단계의 끝에는 반드시 "다음 단계로 가는 버튼"이 있어야 합니다. 유튜브 설명란 → 무료 진단 링크, 블로그 글 마지막 → 해당 진단 신청, 진단 리포트 끝 → 상담 예약. 콘텐츠가 영업을 대신하게 만드는 게 자동화의 출발점입니다.

실행 체크리스트

  • 기존 콘텐츠 자산 목록화 유튜브 영상·저서 챕터·방송 클립·칼럼을 표로 정리하고, 각각을 퍼널 5단계 중 어디에 쓸지 매핑
  • 유튜브 영상 설명란에 무료 진단 링크 삽입 인기 영상 상위 10개부터. "이 영상 보고 궁금하면 → 무료 부지/도면 검토 신청"
  • 저서 구매자 → 상담 연결 경로 만들기 책 안내 페이지 또는 QR로 무료 진단 유입 (독자는 이미 최상위 진성 고객)
  • "상담 전 필독/필시청" 큐레이션 페이지 대표 영상 3개 + 글 3개를 묶어 "상담 전 이것부터 보세요" 코너 → 래포를 상담 전에 완성
  • 각 진단 리포트 마지막에 상담 예약 CTA 고정 진단 → 상담 전환 지점 표준화
  • 퍼널 지표 정의 영상 조회 → 진단 신청 → 상담 → 계약 각 전환율을 측정할 최소 지표 4개 확정
S-2

포지셔닝 통일

"예쁜 단독주택 짓는 회사"로 가면 무면허·소규모 시공사와 평당가 비교에서 구조적으로 불리합니다. 강점은 통합 역량입니다.

확정 포지셔닝: 대지·사업성·디자인·자재·견적·시공·브랜딩을 연결하는 소형 수익형 건축 전문 디자인빌드 종합건설사. 차별점 한 문장: "자재사양서와 자재배치도를 견적 전에 만들고, 그걸 바탕으로 견적을 내는 회사" → 실제 공사 중 변경·추가금을 줄인다.
  • 홈페이지·소개서·제안서·광고의 한 줄 소개 문구 통일 모든 채널에서 같은 포지셔닝 문장을 쓴다
  • 타겟 확장 반영 단독주택 → 상가주택·근생·카페·스테이·세컨하우스·수익형·도면 보유 시공 의뢰로 메시지 확장
  • "견적 전 자재·마감 정리" 차별점을 전 채널 핵심 메시지로 승격 홈페이지엔 이미 반영됨 → 소개서·광고·상담에도 동일하게
W-0

이번 주 바로 할 일

사이트 구조를 바꿀 필요 없이 지금 있는 폼 위에 "상품 레이어"만 얹으면 됩니다. 매출에 가장 가까운 순서입니다.

  • 우선고객 유형 5개 확정 부지 보유형 / 수익형(상가주택·근생) / 카페·스테이 창업 / 도면 보유 시공 의뢰 / 전원주택·세컨하우스
  • 우선무료 진단 상품명 확정 부지 1차 검토 · 도면 기반 견적 전 점검 · 수익형 건축 1차 검토 (세컨·스테이 컨셉 진단은 2차)
  • 문의 폼 재설계(조건부) 현재 단계 선택에 따라 추가 질문 노출 · 아래 AX 섹션 참조
  • 1페이지 회사 소개서 작성 홈페이지보다 먼저 필요한 영업 무기
  • AI 상담 브리핑 양식 만들기 아래 AX 섹션의 프롬프트 템플릿 사용
  • 상담 후속 메시지 템플릿 4종 세팅 부지/도면/수익형/세컨 — 참조 섹션에 초안 있음
W-1

진단 상품화

"건축 상담 문의" 대신 "내 땅/도면/예산으로 가능한 방향을 먼저 진단해드립니다"로 진입 장벽을 낮춥니다. 지금 당장 계약 준비가 안 된 고객도 문의할 이유가 생깁니다.

진단 1

부지 1차 검토 리포트

땅은 있는데 뭘 지을지 모르는 고객. 건축 가능 방향·수익형 가능성·리스크 체크.

진단 2

상가주택·근생 수익성 1차 검토

임대·운영 수익 기대 고객. 용도·공사비 감각·사업성 체크포인트.

진단 3

도면 기반 시공 견적 전 점검

도면은 있는데 견적이 불안한 고객. 빠진 마감·자재·공사 범위 확인.

진단 4

세컨하우스·스테이 컨셉 진단

부지·예산 기준 컨셉·운영 동선·브랜딩 방향 제안.

  • 진단 상품별 "제공 가치 + 고객에게 보이는 문구" 확정 참조 섹션의 광고/문구 모음 활용
  • 진단별 내부 접수 항목 정의 부지: 위치·면적·지목·용도·접도·희망용도·예산·자료·일정
  • 홈 히어로 아래 유형별 진입 카드 3개 배치 "땅은 있는데" / "도면은 있는데" / "사업성부터" → 각 진단 신청 연결
  • 진단 결과 리포트 1페이지 템플릿 제작 진단→상담 전환의 핵심 문서. 마지막에 상담 예약 CTA
  • 진단 3종부터 오픈 · 세컨/스테이는 2차 과하게 벌리지 말고 3개로 시작

AX

AX 자동화 — 문의 이후의 흐름

가장 빠른 ROI 지점입니다. 문의가 오면 AI가 자동으로 1페이지 상담 브리핑을 작성해, 상담 전에 고객 유형·계약 가능성·추가 질문·후속 메시지 초안을 준비합니다. 거창한 시스템 없이, 폼 → 스프레드시트/메일 → 프롬프트로 시작할 수 있습니다.

① 문의 폼 조건부 재설계

  • 폼기본 정보 + 상담 희망 지역/방식(전화·방문·온라인) 수집
  • 폼"현재 단계" 선택에 따른 조건부 질문 부지 보유→면적·지번·용도 / 도면 보유→도면·견적서 파일 업로드 / 수익형→운영 목적
  • 폼파일 업로드 필드 추가 도면 보유 고객이 파일을 못 올리는 게 현재 가장 아쉬운 지점
  • 폼예산·자료 보유·고민 선택지 추가 문의 순간부터 고객 유형이 자동 분류되도록

② 상담 브리핑 자동화

PROMPT상담 브리핑 생성 프롬프트 (폼 데이터 넣고 실행)

홈페이지 개선

리뉴얼은 이미 잘 되어 있습니다(포지셔닝 메시지, 견적 전 자재 정리 문구, 신뢰 요소, 1차 문의 폼). 지금 필요한 건 "진단 상품 + 유형 분기 + 문의 이후 흐름"입니다.

N-1

NAS 라벨링 (흐름에 종속시켜 시작)

2TB 전체를 AI에 통째로 먹이지 않습니다. 원본은 NAS에 두고, AI는 파일의 지도(index)를 먼저 읽습니다. 그리고 라벨링을 별도 프로젝트가 아니라 위 작업(사례 페이지·제안서)의 재료 공급용으로 먼저 돌립니다.

순서: 구글드라이브는 백업 용도로만 OK. Codex/Claude는 통째 읽기가 아니라 라벨링 파이프라인 제작 용도. 실제 시작은 대표 프로젝트 5개의 자재시트·자재배치도·대표 완공사진만 선별 인덱싱.
  • 2TB 구글드라이브 백업 시작(백업 목적 한정)
  • NAS 전체 파일 목록(인덱스)만 먼저 생성 경로·파일명·확장자·크기·수정일·상위폴더
  • 폴더/파일명 기반 1차 자동 분류표 AI가 내용을 읽기 전에 이름만으로 카테고리 분류
  • 대표 프로젝트 5개만 샘플 라벨링(PoC)
  • arkireal-data-indexer 레포 생성 scan_files · classify_paths · extract_excel/pdf · thumbnails
  • 자재시트·자재배치도·완공사진 대표컷부터 DB화 영상 전체 라벨링은 비추천, 색인만

사용법 · 구글시트 연결

세팅 전에도 이 파일은 바로 작동합니다(로컬 저장). 아래 순서로 구글시트에 연결하면, 상태·메모가 시트에 저장되고 PC·휴대폰·직원이 같은 데이터를 공유하게 됩니다.

1단계 · GitHub Pages 배포

  1. github.com 에서 새 저장소 생성 (예: arkireal-dashboard)
  2. 이 index.html 업로드 (Add file → Upload files)
  3. Settings → Pages → Branch: main / (root) → Save
  4. 1~2분 뒤 https://아이디.github.io/arkireal-dashboard/ 접속

2단계 · 구글시트 + Apps Script

  1. 구글 드라이브에서 새 구글 시트 생성
  2. 상단 메뉴 확장 프로그램 → Apps Script
  3. 기본 코드를 지우고 아래 코드를 붙여넣고 저장(디스크 아이콘)
  4. 우측 상단 배포 → 새 배포 → 유형: 웹 앱
  5. 실행: 나 / 액세스 권한: 모든 사용자 → 배포 → (최초 1회 권한 승인)
  6. 나온 웹 앱 URL(끝이 /exec)을 복사
APPS SCRIPT시트에 붙여넣을 코드 (TOKEN은 대시보드와 동일하게)
var TOKEN = "arkireal-secret-1234";
var SHEET = "data";

function sh_(){
  var ss = SpreadsheetApp.getActiveSpreadsheet();
  var s = ss.getSheetByName(SHEET);
  if(!s){ s = ss.insertSheet(SHEET); s.appendRow(["id","status","memo","updated"]); }
  return s;
}
function out_(o){
  return ContentService.createTextOutput(JSON.stringify(o)).setMimeType(ContentService.MimeType.JSON);
}
function doGet(e){
  if(e.parameter.token !== TOKEN) return out_({ok:false});
  var v = sh_().getDataRange().getValues();
  var items = {};
  for(var i=1;i<v.length;i++){
    if(v[i][0]==="") continue;
    items[String(v[i][0])] = {s:v[i][1]||"todo", m:v[i][2]||""};
  }
  return out_({ok:true, items:items});
}
function doPost(e){
  var b = {};
  try{ b = JSON.parse(e.postData.contents); }catch(err){}
  if(b.token !== TOKEN) return out_({ok:false});
  var s = sh_(), last = s.getLastRow();
  if(b.action === "reset"){
    if(last > 1) s.deleteRows(2, last-1);
    return out_({ok:true});
  }
  var ids = last===0 ? [] : s.getRange(1,1,last,1).getValues();
  var row = 0;
  for(var i=1;i<ids.length;i++){ if(String(ids[i][0])===String(b.id)){ row=i+1; break; } }
  if(row===0) s.appendRow([b.id, b.s||"todo", b.m||"", new Date()]);
  else s.getRange(row,2,1,3).setValues([[b.s||"todo", b.m||"", new Date()]]);
  return out_({ok:true});
}

3단계 · 대시보드에 연결

  1. 이 대시보드 좌측 하단 ⚙ 시트 연결 클릭
  2. 웹 앱 URL과 토큰(arkireal-secret-1234) 입력 → 연결 테스트 → 저장하고 동기화
  3. 사이드바 표시가 "시트 동기화됨"(초록)으로 바뀌면 완료
모든 기기에 기본 적용: 위 3단계는 그 기기에만 적용됩니다. PC·휴대폰·직원 모두 자동 연결하려면, index.html 상단 스크립트의 CFG_URL_DEFAULT·CFG_TOKEN_DEFAULT 두 줄을 실제 값으로 바꿔 다시 업로드하세요. 그러면 접속하는 모든 기기가 자동으로 시트와 연결됩니다.
주의: ① 동시 편집은 "나중에 저장한 것이 이김"입니다(혼자/소수면 문제없음). ② 토큰은 코드의 TOKEN과 대시보드 값이 반드시 같아야 합니다. ③ 토큰은 접근을 막는 최소 장치일 뿐이니, 민감정보는 넣지 마세요. ④ 오프라인이거나 시트 연결 실패 시 자동으로 로컬 저장으로 유지되고, 다시 연결되면 동기화됩니다.
